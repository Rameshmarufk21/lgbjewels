LabGrownBox Deployment Playbook

This is the exact sequence to take the app from local machine to production.

0) Already completed

  • Migration status checked; schema up to date.
  • npx prisma migrate deploy executed (no pending migrations).
  • npm run build executed successfully.

1) Accounts needed (one-time)

  1. GitHub
  2. Vercel
  3. Neon or Supabase (Postgres)
  4. Google AI Studio (Gemini key)
  5. Optional: Google Apps Script account (for Sheets sync)

2) Push code to GitHub

git status
git add .
git commit -m "prepare production-ready deployment"
git remote add origin <your-github-repo-url>   # only if not set
git push -u origin main

3) Create production Postgres

Create DB in Neon/Supabase and copy DATABASE_URL.

postgresql://USER:PASSWORD@HOST/DB?sslmode=require

4) Connect project in Vercel

  1. Import GitHub repo in Vercel
  2. Framework: Next.js
  3. Add env vars:
    • DATABASE_URL
    • GEMINI_API_KEY
    • Optional GEMINI_MODEL

5) Apply production migrations

DATABASE_URL="<your-prod-postgres-url>" npx prisma migrate deploy

6) First production validation

  1. Open / and verify orders UI loads
  2. Open /settings
  3. Test Gemini key in settings
  4. Scan one invoice
  5. Create/edit/delete an order
  6. Check dashboard and statements

7) Optional Sheets sync

  1. Create Apps Script Web App URL
  2. Set script URL + shared secret in app settings
  3. Save one order and verify sheet update

8) Storage warning

Current app can use local/server filesystem for some assets. This is not durable on serverless hosting. For robust production, move uploads to cloud object storage (S3/Supabase Storage/Vercel Blob).

9) Daily update flow

git add .
git commit -m "your change"
git push

Vercel auto-redeploys from GitHub.
